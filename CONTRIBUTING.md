# Contributing to Career Compass

Thank you for your interest in contributing to Career Compass! This document provides guidelines and instructions for contributing.

## CodeRabbit Integration

This project uses **CodeRabbit** for automated code reviews. All pull requests are automatically reviewed for:

- Code quality and best practices
- Security vulnerabilities
- Documentation completeness
- Performance optimizations

## Pull Request Process

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/amazing-feature`
3. **Make your changes**
4. **Ensure CodeRabbit passes**: Check the PR for CodeRabbit review comments
5. **Update documentation**: If needed, update README or other docs
6. **Commit your changes**: `git commit -m 'Add amazing feature'`
7. **Push to branch**: `git push origin feature/amazing-feature`
8. **Open a Pull Request**

## Code Style

- Follow existing code style
- Add comments for complex logic
- Write self-documenting code
- Include docstrings for functions

## Testing

- Add tests for new features
- Ensure all tests pass
- Update tests when fixing bugs

## Documentation

- Update README if adding features
- Add JSDoc/Python docstrings
- Include examples in documentation

## CodeRabbit Review Checklist

Before submitting a PR, ensure:

- [ ] Code follows project conventions
- [ ] No security vulnerabilities
- [ ] Documentation is updated
- [ ] Tests are included (if applicable)
- [ ] CodeRabbit review is addressed

## Questions?

Open an issue or contact the maintainers.

---

**Note**: CodeRabbit will automatically review your PR and provide feedback. Please address any concerns before requesting a merge.
